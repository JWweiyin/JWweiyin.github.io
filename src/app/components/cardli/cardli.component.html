About CardLI


CardLI is a Command line Interface (CLI) desktop app that helps students manage their flashcards. It also allows students to test and review their knowledge. All of this in one single platform.

CardLI provides a user-friendly, storage efficient and internet connection independent flashcard experience to students in institutes of higher education, who are able to type fast and are reasonably comfortable using CLI apps.

My contributions to this team project were:

  • Implementing the function to add a new deck of flashcards to the program
  • Implementing the function to add a new flashcard to a deck of flashcards
  • Implementing a search system that returns flashcards matching the search term to the user

Development Process


The application was developed in two iterative stages, or milestones.


Milestone 1


Milestone 1 featured:

  • Usage instructions for the user to refer to if they forget how to use the application
  • A function to add flashcards
  • A function to delete flashcards
  • A function to view flashcards
  • A test function that allowed the user to test themselves with their flashcards
  • A function to display the list of incorrectly answered flashcards upon the conclusion of a round of testing

Milestone 2


Along with all the functionalities of Milestone 1, Milestone 2 featured:


  • A function to shuffle flashcards for testing
  • A function to edit flashcards
  • Functions to add, edit and delete decks that would serve as a set to keep flashcards with related topics in
  • A timer for test mode
  • Test statistics for each flashcard
  • A review function that would allow the user to test themselves on flashcards they answered incorrectly in more than half the total attempts
  • A search function to find flashcards that matched the search terms
  • An automatic save system that saved the state of the cards after each action

User Testing


System Testing was carried out by a team of testers to spot bugs in the application. These bugs were reported as issues as GitHub, following which my team took steps to squash before the final release of the application.

Learning Points


Working on this project significantly improved on my understanding of software engineering, documentation writing and version control. Though it still should be said that the project was not without its hiccups. The structure of our program underwent major changes even after the delivery of Milestone 1 as the my team was unsure of what structure would be the most organised and efficient. To avoid this situation we should have planned out the structure as a group before beginning to implement the program as being clear of the overall structure would allow us to write code that could be systematically and modularly implemented into the program.
